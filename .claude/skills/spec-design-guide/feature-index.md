# Unison Flow 機能インデックス

このファイルは、spec/ディレクトリ内の各機能に割り当てられた連番を管理します。

## 連番の割り当て

| 連番 | 機能名 | ディレクトリ | 状態 | 説明 |
|------|--------|------------|------|------|
| 01 | Core Concept | `spec/01-core-concept/` | 作成済 | Unison Flowの基本コンセプト・ビジョン |
| 02 | KDL Parser | `spec/02-kdl-parser/` | 作成済 | KDLファイルのパース機能 |
| 03 | Runtime Strategy | `design/03-runtime/` | 作成済 | コンテナランタイム戦略（Docker/K8s/Cloud Run） |
| 04 | Service Management | - | 未定 | サービス定義と管理 |
| 05 | CLI Interface | - | 未定 | `flow` コマンドのCLIインターフェース |
| 06 | Stage Management | - | 未定 | ステージ（local/dev/stg/prd）の管理 |
| 07 | Container Operations | - | 未定 | コンテナの起動・停止・削除操作 |
| 08 | Volume Management | - | 未定 | ボリュームマウント管理 |
| 09 | Network Configuration | - | 未定 | ネットワーク・ポート設定 |
| 10 | Environment Variables | - | 未定 | 環境変数の管理 |

## 連番割り当てルール

- **01-09**: コア機能
- **10-19**: 基本機能
- **20-29**: 拡張機能
- **30-39**: 統合・連携機能
- **40-49**: ツール・ユーティリティ
- **50-99**: 将来の拡張用予約

## 新機能追加時のワークフロー

1. このファイルで次の連番を確認
2. `spec/{連番}-{機能名}/` ディレクトリを作成
3. SPEC.md（コンセプト・仕様・哲学）を作成
4. DESIGN.md（モデル・手法・実装）を作成
5. このファイルを更新（状態を「作成済」に変更）

## 注意事項

- 連番は欠番にしない（将来的な整理のため）
- ディレクトリ名は `{連番}-{kebab-case}` 形式
- 状態: `未定` → `作成済` → `実装中` → `完了`
